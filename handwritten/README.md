# RetroQuest Mini Compiler

RetroQuest is a retro-style mini compiler project for Compiler Construction. It implements a custom DSL for text adventures and interactive fiction. The compiler performs lexical analysis, parsing, semantic analysis, IR generation, basic optimization, and code execution through an interpreter.

## Features
- Custom source language with `.rq` extension
- Lexer with line/column aware error reporting
- Parser that builds an AST from RetroQuest source
- Semantic analyzer with variable and scene validation
- Custom IR generation for story graphs
- Two optimization passes:
  - constant folding and propagation
  - unreachable code and unreachable scene elimination
- Command-line execution and REPL-style interactive mode
- Debug mode to inspect compiler phases

## Requirements
- Python 3.10 or later

## Usage
Compile a source file:
```bash
python main.py examples/treasure_hunt.rq
```

Compile and write optimized IR:
```bash
python main.py examples/treasure_hunt.rq -o output/story.json
```

Show compiler phases:
```bash
python main.py examples/treasure_hunt.rq --debug
```

Compile and run:
```bash
python main.py examples/treasure_hunt.rq --run
```

Start REPL mode:
```bash
python main.py --interactive
```

## REPL Commands
- `:run` compiles and runs the current buffer
- `:show` prints the current buffer
- `:reset` clears the current buffer
- `:quit` exits the REPL

## Output
RetroQuest can produce:
- optimized JSON IR
- interactive execution in the terminal
- debug output for tokens, AST, symbol table, and IR before/after optimization

## Manual Deliverables Still Needed
The semester brief also requires handwritten artifacts. These are not auto-generated by the compiler and should be prepared separately:
- DFA or transition table
- 2 handwritten parse trees
- handwritten symbol table example with scope view

## Suggested Demo Flow
1. Show a valid `.rq` source file
2. Run `--debug` to explain compiler phases
3. Export optimized IR with `-o`
4. Run the same file with `--run`
5. Run an invalid file to demonstrate error handling
